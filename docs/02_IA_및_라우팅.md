# 02. IA 및 라우팅 설계

## 1. 사이트맵

```text
INSHOW ACADEMY
├─ Home
├─ Courses
│  ├─ 전체 강의
│  ├─ 입문 실무
│  ├─ 도면/설계
│  ├─ 견적/계약
│  ├─ 현장관리
│  └─ 창업/브랜딩
├─ Course Detail
├─ Checkout
│  ├─ 신청 정보 입력
│  ├─ 결제
│  ├─ 결제 성공
│  └─ 결제 실패
├─ Auth
│  ├─ 로그인
│  └─ 회원가입
├─ My Page
│  ├─ 신청 강의
│  ├─ 결제 내역
│  ├─ 강의 자료
│  └─ 환불 요청
├─ Support
│  ├─ FAQ
│  └─ 문의하기
└─ Admin
   ├─ Dashboard
   ├─ Courses
   ├─ Enrollments
   ├─ Orders
   ├─ Refunds
   └─ Notices
```

## 2. 라우트 테이블

| Path | Page Component | 설명 |
|---|---|---|
| `/` | `HomePage` | 브랜드 소개, 대표 강의, 강의 CTA, 후기, FAQ 요약 |
| `/courses` | `CourseListPage` | 전체 강의 목록, 카테고리 필터, 정렬, 마감 상태 표시 |
| `/courses/:slug` | `CourseDetailPage` | 강의 상세, 커리큘럼, 일정 선택, sticky 신청 박스 |
| `/checkout/:courseId` | `CheckoutPage` | 신청자 정보, 일정 선택 확인, 주문 생성, 결제 위젯 호출 |
| `/payment/success` | `PaymentSuccessPage` | PG 성공 리다이렉트 후 서버 승인/검증 처리 |
| `/payment/fail` | `PaymentFailPage` | 결제 실패 코드/메시지 안내 및 재시도 |
| `/login` | `LoginPage` | 이메일 로그인 및 소셜 로그인 진입 |
| `/signup` | `SignupPage` | 회원가입, 약관/개인정보 동의 |
| `/mypage` | `MyPage` | 신청 강의, 결제 내역, 자료 다운로드, 환불 요청 |
| `/support/faq` | `FaqPage` | 수강/결제/환불/자료 관련 FAQ |
| `/support/contact` | `ContactPage` | 문의 폼 및 채널 안내 |
| `/admin` | `AdminDashboardPage` | 관리자 대시보드, 핵심 지표 |
| `/admin/courses` | `AdminCourseListPage` | 강의 등록/수정/비공개/정원 관리 |
| `/admin/enrollments` | `AdminEnrollmentPage` | 신청자, 결제 상태, 출석/수강 상태 관리 |
| `/admin/orders` | `AdminOrderPage` | 주문/결제/환불 조회 |


## 3. 라우팅 원칙

- 공개 페이지와 관리자 페이지는 레이아웃을 분리한다.
- 결제 성공/실패 페이지는 PG 리다이렉트 URL로 사용되므로 직접 접근에도 안전하게 처리한다.
- 관리자 라우트는 role이 admin인 사용자만 접근 가능하게 한다.
- 강의 상세는 `/courses/:slug` 구조로 검색과 공유에 유리하게 설계한다.
