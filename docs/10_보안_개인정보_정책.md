# 10. 보안/개인정보/운영 정책

## 1. 인증/인가

- access token + refresh token 또는 secure session 기반 인증을 선택한다.
- 관리자 라우트와 API는 role 검사를 필수로 한다.
- 관리자 계정은 2단계 인증을 검토한다.

## 2. 개인정보

- 수집 항목을 최소화한다.
- 개인정보 처리방침, 이용약관, 환불 규정 페이지를 제공한다.
- 신청 정보와 결제 정보 접근은 본인 및 관리자만 가능해야 한다.
- CSV 다운로드는 관리자만 가능하며 다운로드 로그를 남긴다.

## 3. 결제 보안

- secret key는 서버 환경변수에만 저장한다.
- 결제 금액 검증은 서버 DB 기준으로 한다.
- 웹훅 검증, idempotency 처리, 중복 승인 방지를 구현한다.
- 결제 상태는 pending/paid/failed/refunded 등 명확한 상태값으로 관리한다.

## 4. 프론트 보안

- XSS 방지를 위해 HTML 삽입을 제한한다.
- 관리자 에디터에서 rich text를 사용할 경우 sanitize 처리한다.
- API 에러 메시지는 내부 정보를 노출하지 않는다.
- 환경변수 중 브라우저에 노출되는 값과 서버 secret을 분리한다.

## 5. 백업/로그

- 주문/결제/환불 데이터는 정기 백업한다.
- 결제 승인 실패, 웹훅 실패, 환불 실패는 알림 대상이다.
- 운영 로그는 최소 1년 보관을 검토한다.
