# 05. 기능 명세

## 1. 기능 요구사항 목록

| ID | 단계 | 요구사항 | 검수 기준 |
|---|---|---|---|
| BR-01 | MVP | 방문자는 강의 목록과 상세 정보를 모바일/데스크톱에서 확인할 수 있어야 한다. | 강의 목록 API 연결, 상세 페이지 slug 라우팅, 반응형 검수 |
| BR-02 | MVP | 사용자는 강의 일정/회차를 선택하고 수강 신청을 할 수 있어야 한다. | 정원 초과 방지, 신청자 정보 검증, 약관 동의 저장 |
| BR-03 | MVP | 사용자는 PG 결제창을 통해 카드/계좌이체 등 결제를 진행할 수 있어야 한다. | 주문 생성 후 결제, success/fail URL 처리, 서버 승인 |
| BR-04 | MVP | 서버는 결제 성공 후 paymentKey/orderId/amount를 검증하고 DB 상태를 paid로 변경해야 한다. | 금액 위변조 방지, 중복 승인 방지, 실패 시 rollback |
| BR-05 | MVP | 사용자는 마이페이지에서 신청 내역과 결제 내역을 확인할 수 있어야 한다. | 주문 상태, 강의 일정, 결제 수단, 영수증 링크 |
| BR-06 | MVP | 관리자는 강의, 커리큘럼, 일정, 정원, 가격을 관리할 수 있어야 한다. | CRUD, 공개/비공개, 마감 처리 |
| BR-07 | MVP | 관리자는 신청자와 결제 상태를 조회하고 환불 요청을 처리할 수 있어야 한다. | 상태 필터, CSV 내보내기, 환불 승인/거절 기록 |
| BR-08 | M1 | 관리자는 쿠폰을 발급하고 특정 강의/기간에 적용할 수 있어야 한다. | 정액/정률, 사용 제한, 중복 사용 방지 |
| BR-09 | M1 | 강의 자료를 수강생에게만 제공할 수 있어야 한다. | 권한 검증, 다운로드 로그, 기간 제한 |
| BR-10 | M2 | 온라인 영상 강의, 진도율, 수료증 기능을 확장할 수 있어야 한다. | LMS 모듈 분리 설계, 추후 VOD 연동 |


## 2. 홈

- Hero: 브랜드 카피, 강의 CTA, 대표 이미지 또는 추상적인 흑백 그래픽
- Featured Courses: 대표 강의 3-6개
- Why INSHOW: 실무 중심 강의의 차별점
- Curriculum Flow: 이해 -> 설계 -> 견적 -> 현장 -> 상담 -> 성장
- Reviews: 후기 카드
- Final CTA: 전체 강의 보기

## 3. 강의 목록

- 필터: 전체, 입문, 도면/설계, 견적/계약, 현장관리, 창업/브랜딩
- 정렬: 최신순, 마감임박순, 가격순
- 표시: 강의명, 요약, 카테고리, 일정, 가격, 마감 상태
- Empty State: 조건에 맞는 강의가 없을 때 안내

## 4. 강의 상세

- 상단: 강의명, 한 줄 설명, 핵심 혜택, 가격, 일정
- 본문: 커리큘럼, 수강 대상, 제공 자료, 강사 소개, FAQ, 환불 규정
- 신청 박스: 일정 선택, 정원, 가격, 수강 신청 CTA
- 모바일: 하단 fixed CTA

## 5. 수강 신청/결제

- 신청자 정보 입력
- 약관/개인정보 동의
- 주문 생성
- PG 결제창 호출
- 성공 시 서버 승인 검증
- 실패 시 사유 안내 및 재시도

## 6. 마이페이지

- 신청 강의 목록
- 결제 내역
- 결제 영수증/확인 정보
- 강의 자료 다운로드
- 환불 요청

## 7. 관리자

- 대시보드: 매출, 신청 수, 결제 성공률, 마감임박 강의
- 강의 관리: 등록/수정/삭제 또는 비공개
- 일정 관리: 회차, 장소, 정원, 신청 인원
- 신청자 관리: 상태 변경, 메모, CSV 내보내기
- 결제/환불 관리: 결제 상세, 환불 승인, 처리 로그
- 공지/FAQ 관리
