# 08. 결제/신청/환불 플로우

## 1. 결제 기본 흐름

```text
강의 상세
-> 일정 선택
-> 로그인 또는 회원가입
-> 신청자 정보 입력
-> 주문 생성 pending
-> PG 결제창 호출
-> 결제 성공 리다이렉트
-> 서버 결제 승인/금액 검증
-> 주문 paid 변경
-> 수강 신청 confirmed 변경
-> 완료 페이지/이메일/문자 안내
```

## 2. 중요 보안 원칙

- PG secret key는 서버에만 저장한다.
- 프론트에서 받은 amount를 신뢰하지 않는다.
- 서버 DB의 주문 금액과 PG 승인 요청 금액을 반드시 비교한다.
- successUrl 접근만으로 결제를 완료 처리하지 않는다.
- 웹훅은 중복 수신될 수 있으므로 idempotent하게 처리한다.

## 3. 결제 실패 흐름

- PG failUrl로 이동
- errorCode, errorMessage 표시
- 동일 주문 재시도 가능 여부 확인
- 재시도 시 새 주문 생성 또는 기존 pending 주문 재사용 정책 결정

## 4. 환불 흐름

```text
사용자 환불 요청
-> 관리자 확인
-> 환불 가능 여부/규정 확인
-> PG 취소 API 호출
-> refunds 상태 completed
-> orders 상태 refunded 또는 partial_refunded
-> enrollments 상태 cancelled
-> 수강생 안내
```

## 5. 환불 정책 화면에 반드시 포함할 항목

- 강의 시작 전 환불 가능 기간
- 강의 시작 후 환불 가능 여부
- 자료 다운로드 후 환불 제한 여부
- PG 수수료 또는 부분 환불 정책
- 문의 채널

## 6. 운영 알림

- 결제 성공: 사용자 이메일/SMS, 관리자 알림
- 결제 실패: 사용자 재시도 안내
- 환불 요청: 관리자 알림
- 환불 완료: 사용자 알림
