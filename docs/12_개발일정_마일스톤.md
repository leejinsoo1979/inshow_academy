# 12. 개발 일정 및 마일스톤

## 1. 권장 일정: 6주 MVP

| 주차 | 목표 | 산출물 |
|---|---|---|
| 1주차 | 요구사항 확정, IA, 디자인 시스템 | 화면 목록, 라우팅, 디자인 토큰, API 초안 |
| 2주차 | 퍼블릭 화면 개발 | 홈, 강의 목록, 강의 상세, 반응형 UI |
| 3주차 | 인증/신청/주문/결제 개발 | 로그인, 신청 폼, 주문 생성, 결제 성공/실패 |
| 4주차 | 마이페이지/관리자 MVP | 신청 내역, 결제 내역, 강의/신청자 관리 |
| 5주차 | QA/보안/운영 기능 | 환불, 알림, 로그, 오류 처리, 테스트 |
| 6주차 | 스테이징/UAT/런칭 | 실결제 전환, 운영자 교육, 배포 체크리스트 |

## 2. 마일스톤

### M0 - 기획 확정

- 서비스 범위 확정
- 강의 카테고리/가격/환불정책 확정
- PG사 선택
- 관리자 권한 정책 확정

### M1 - 퍼블릭 베타

- 홈/강의 목록/상세 개발 완료
- 브랜드 UI 검수 완료
- 모바일 반응형 완료

### M2 - 결제 MVP

- 주문 생성
- PG 테스트 결제
- 결제 승인 검증
- 신청 확정

### M3 - 운영 MVP

- 관리자 강의/신청/결제 관리
- 환불 처리
- 알림/로그

### M4 - 런칭

- 실결제 키 전환
- 운영자 테스트 완료
- 최종 QA 완료

## 3. 착수 전 확정해야 할 질문

- 오프라인 강의인지, 온라인 라이브인지, VOD인지?
- 강의 정원과 마감 정책은 어떻게 할 것인지?
- 환불 규정은 어떻게 적용할 것인지?
- 세금계산서/현금영수증 발행이 필요한지?
- 로그인은 이메일만 할지, 카카오/네이버를 포함할지?
- 관리자 수는 몇 명이며 권한을 나눌지?
